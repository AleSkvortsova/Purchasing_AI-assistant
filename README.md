# Procurement Intake Assistant

Минимальный каркас сервиса для будущего ИИ-ассистента по внутренним заявкам
на закупку. Приложение предоставляет базовые служебные HTTP endpoints.

## Текущий статус

Реализован стартовый FastAPI-сервис версии `0.1.0`, конфигурация из окружения,
стандартное логирование и автоматические тесты health-check.

Telegram, OpenAI API, Supabase, PostgreSQL, pgvector, RAG, n8n, Docker Compose и
бизнес-логика заявок на этом этапе не подключены.

## Границы MVP

Утверждённые продуктовые границы и критерии готовности описаны в
[MVP_SCOPE.md](MVP_SCOPE.md). Этот репозиторий пока содержит только технический
фундамент для последующих этапов MVP.

## Требования

- Python `>=3.11,<3.13`;
- Git.

## Установка

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -e ".[dev]"
```

Скопируйте `.env.example` в `.env`, только если нужно переопределить настройки.
Файл `.env` не должен попадать в Git.

## Локальный запуск

```powershell
uvicorn app.main:app --reload
```

Сервис будет доступен по адресу `http://127.0.0.1:8000`.

## Проверка health

```powershell
Invoke-RestMethod http://127.0.0.1:8000/health
Invoke-RestMethod http://127.0.0.1:8000/api/v1/health
```

Оба запроса возвращают:

```json
{"status": "ok"}
```

## Запуск тестов

```powershell
pytest
```

## Проверка Ruff

```powershell
ruff check .
```

## Структура проекта

```text
app/
├── api/             # HTTP routers
├── core/            # настройки и логирование
├── bot/             # будущая Telegram-интеграция
├── llm/             # будущая LLM-интеграция
├── rag/             # будущий поиск по базе знаний
├── repositories/    # будущие адаптеры хранения
├── schemas/         # схемы данных
├── services/        # прикладные сервисы
└── main.py          # точка входа FastAPI
docs/                # проектная документация и база знаний
prompts/             # будущие шаблоны промптов
scripts/             # служебные скрипты
tests/               # автоматические тесты
MVP_SCOPE.md         # утверждённые границы MVP
pyproject.toml       # зависимости и настройки инструментов
```

## Следующие этапы

После отдельного согласования можно перейти к моделированию домена заявки и
контрактов между слоями. Интеграции и бизнес-логика должны добавляться
последовательно в рамках `MVP_SCOPE.md`.
